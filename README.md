# Fort Lupton Lumber Mill Website

## Overview

This repository contains the JSON-based content structure and planning documentation for the Fort Lupton Lumber Mill website. The site will be deployed using HTML YMSL or JS and serves both informational and e-commerce purposes.

## Business Information

- **Business Name:** Fort Lupton Lumber Mill
- **Location:** Fort Lupton, Colorado
- **Primary Goals:** Provide information about lumber products and services; enable online ordering of shavings
- **Target Audience:** Local farms, construction businesses, contractors, agricultural operations

## Repository Structure

```
.
├── README.md           # This file - project overview and setup guide
├── NOTES.md            # Detailed requirements, success criteria, and project notes
├── plan.json           # Structured task breakdown across Phases 0-4
├── site config         # Basic site configuration (colors, theme, navigation)
└── (future files)      # schema.json, page-specific JSON, Wix integration guide
```

## Required Pages

1. **Home** - Introduction, hero section, key offerings
2. **About** - Company history, values, team, facilities
3. **Services** - Detailed service offerings and capabilities
4. **Order Shavings Now** - E-commerce functionality for shavings products
5. **FAQ** - Common questions about products, delivery, pricing
6. **Contact** - Contact form, location map, business hours

## Tech Stack

- **Content Format:** JSON schemas defining page structure and content
- **Hosting Platform:** Wix
- **Version Control:** Git/GitHub for content management

## Getting Started

This is a content repository for a Wix-based website. To use this content:

1. **Review** `NOTES.md` for detailed requirements and success criteria
2. **Examine** `plan.json` for the complete task breakdown
3. **Check** `site config` for basic theme and navigation structure
4. **Follow** the implementation guide (to be created in Phase 3) for translating JSON content into Wix pages

## Development Workflow

1. All content changes should be committed to this repository
2. Use conventional commit messages (feat:, docs:, fix:, etc.)
3. Update `plan.json` task statuses as work progresses
4. Update `NOTES.md` with any new decisions or open questions
